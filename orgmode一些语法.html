orgmode只是提供了一个很好的工具，但能否用好这个工具才是重点。可以用兵器和招式来理解。官方文档上说可以做为keep notes,todo list,project planning

1 记录一下使用orgmode都可以做哪些事情，怎么做法

  • 记笔记，最基本的
  • Todo list
  • 习惯养成
  • 做项目计划
  • RSS订阅
  • 书签管理
  • 做一个计时器

2 查看和检查org语法树

M-: (org-element-parse-buffer) RET 查看org语法树 org-lint命令可以检查org语法

3 #+开头的是个什么鬼？

以#+开头加关键字和冒号的语法称之为in-buffer settings

#+ 开头的关键字并非是任意写的，orgmode识别这些是有限的，通过输入

#+然后再入M-TAB 自动补全可以看到orgmode支持的关键字，另外当输入了这些关键字之后，可以通过C-c C-c来刷新生效

4 输入结构块的快捷方法

<s TAB completes the ‘src’ code block. <e TAB 输入例子块，这个我常用其他如下： s #+BEGINSRC … #+ENDSRC e #+BEGINEXAMPLE … #+ENDEXAMPLE q #+BEGINQUOTE … #+ENDQUOTE v #+BEGINVERSE … #+ENDVERSE c #+BEGINCENTER … #+ENDCENTER l #+BEGINEXPORT latex … #+ENDEXPORT L #+LATEX: h #+BEGINEXPORT html … #+ENDEXPORT H #+HTML: a #+BEGINEXPORT ascii … #+ENDEXPORT A #+ASCII: i #+INDEX: line I #+INCLUDE: line

5 Drawer

C-c C-x d 添加一个Drawer C-c C-z 这个命令很好使，添加一条日志，包含时间戳和笔记，很适合用来做批注，或者做阅读笔记。如果设置了org-log-into-drawer，会记录在指定的 Drawer中，一般设置为LOGBOOK

6 org-mode的时间戳

默认时间戳是会根据loacl变化的，尤其是星期几，中文就是三，四这样这样，但是公司电脑和个人电脑似乎local不太一样，一个是中文一个是英文。做如下设置，统一使用英文 

(setq system-time-locale "C")

其他local的设置就先不管了，似乎公司电脑用的还是GB2312，语言和local环境设置是另外一个问题了。上面这个方法是在一下地址找到的 https://emacs.stackexchange.com/questions/22371/org-mode-timestamps-encoding
